Seção 1

Pré-Requisitos

Antes de começar a instalação, certifique-se de que você tem em mãos tudo o que é necessário. Separe esses itens antes de ir até o cliente.

Conta Google (Gmail) do cliente
Será usada para criar o projeto Firebase. O cliente precisa fornecer login e senha.
Chrome ou Edge atualizado
O sistema funciona melhor nesses navegadores. Verifique se está atualizado.
Acesso à internet
Necessário para criar o Firebase e para o sistema funcionar após a instalação.
Pen drive com os arquivos do sistema
Contém todos os arquivos do DRG-Kronos 3.0, incluindo o firebase-config.js e o ABRIR.bat.
Importante: Cada cliente precisa de um projeto Firebase próprio. Nunca compartilhe as credenciais de um cliente com outro.
Seção 2

Criar o Projeto Firebase

O Firebase é o banco de dados em nuvem onde os dados do sistema ficam armazenados. Você precisa criar um projeto gratuito para cada cliente.

  1. Acesse o console do Firebase Abra o Chrome e vá para: https://console.firebase.google.com. Entre com a conta Google do cliente.
  2. Clique em "Criar um projeto" Para o nome do projeto, use algo que identifique o cliente, por exemplo: nome-empresa-kronos (sem espaços, sem acentos).
  3. Desative o Google Analytics Na próxima tela, desmarque a opção do Google Analytics — isso simplifica a criação. Clique em "Criar projeto".
  4. Aguarde a criação do projeto O Firebase leva cerca de 30 segundos. Quando aparecer "Seu novo projeto está pronto", clique em "Continuar".
  5. No menu lateral, clique em "Firestore Database" Em seguida clique em "Criar banco de dados".
  6. Selecione "Iniciar no modo de produção" Depois vamos configurar as regras de acesso na Seção 4.
  7. Escolha a região: southamerica-east1 (São Paulo) Essa é a região mais próxima do Brasil, garantindo melhor velocidade. Clique em "Ativar".
  8. Aguarde o provisionamento O banco de dados leva cerca de 1 minuto para ficar disponível. Quando aparecer a tela do Firestore, está pronto.
O Firebase tem plano gratuito. Para o uso típico do DRG-Kronos com até algumas dezenas de colaboradores, o plano gratuito (Spark) é suficiente. Não é necessário cadastrar cartão de crédito.
Seção 3

Obter as Credenciais Firebase

As credenciais são um conjunto de chaves que identificam o projeto Firebase do cliente. Você vai copiá-las e colar no arquivo de configuração do sistema.

  1. Clique na engrenagem ⚙ no topo do menu lateral Selecione "Configurações do projeto".
  2. Role a página até "Seus apps" Na seção "Seus apps", clique no ícone </> (Web app — aplicativo web).
  3. Dê um nome para o app Digite drg-kronos e clique em "Registrar app". Não é necessário ativar o Firebase Hosting.
  4. Copie o objeto firebaseConfig Uma caixa de código aparece na tela com 6 campos (apiKey, authDomain, projectId, storageBucket, messagingSenderId, appId). Copie ou anote todos esses valores.
  5. Clique em "Continuar para o console" As credenciais ficam salvas no projeto — você pode acessá-las novamente pelas Configurações do projeto quando precisar.
Guarde as credenciais em local seguro. O arquivo firebase-config.js de cada cliente é único. Mantenha uma cópia em local seguro para consulta futura.
Seção 4

Configurar as Regras do Firestore

As regras do Firestore controlam quem pode ler e gravar dados no banco. Para o funcionamento inicial do sistema, configure regras abertas conforme descrito abaixo.

  1. No menu lateral do Firebase, clique em "Firestore Database"
  2. Clique na aba "Regras" No topo da tela do Firestore.
  3. Substitua TODO o conteúdo da caixa de texto pelas regras abaixo Apague o que estiver lá e cole exatamente o seguinte: 
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
  1. Clique em "Publicar" As regras são salvas e entram em vigor imediatamente.
Nota sobre segurança: As regras abertas acima são suficientes para a implantação inicial. O sistema possui controle de acesso próprio (usuário e senha). Para ambientes que exijam maior segurança, entre em contato com o suporte técnico para configurar regras mais restritas.
Seção 5

Instalar o Sistema no Computador do Cliente

Com as credenciais Firebase em mãos, agora é hora de copiar os arquivos do sistema para o computador do cliente e configurar a conexão com o banco de dados.

  1. Conecte o pen drive ao computador do cliente
  2. Crie uma pasta no computador do cliente Sugestão: C:\DRG-Kronos\. Pode ser em qualquer local, desde que seja de fácil acesso.
  3. Copie TODOS os arquivos do pen drive para essa pasta Selecione tudo (Ctrl+A) na pasta do pen drive e copie para a pasta criada no computador.
  4. Abra o arquivo firebase-config.js com o Bloco de Notas Clique com o botão direito no arquivo → "Abrir com" → "Bloco de Notas".
  5. Substitua os valores de configuração O arquivo contém placeholders como COLE_AQUI_API_KEY, COLE_AQUI_PROJECT_ID, etc. Substitua cada um pelo valor correspondente copiado do Firebase na Seção 3.
  6. Salve o arquivo Pressione Ctrl+S. Feche o Bloco de Notas.
  7. Dê um duplo clique em ABRIR.bat para testar O sistema deve abrir automaticamente no navegador padrão.
  8. Verifique se a tela de login aparece corretamente Se aparecer a tela de login, a instalação básica está concluída.

Seção 6

Configurar os Dados da Empresa

Após o primeiro login, configure os dados da empresa do cliente para que apareçam corretamente em toda a interface e nos documentos impressos.

  1. Faça login com o usuário admin Use as credenciais da Seção 5.
  2. No menu lateral, clique em "⚙ Configurações" Essa opção aparece apenas para o perfil master.
  3. Preencha os campos com os dados da empresa do cliente
    • Nome da empresa
    • CNPJ
    • Ramo de atividade / Descrição
    • URL do logo (opcional — se não tiver, o sistema usa o logo padrão)
  4. Clique em "Salvar Configurações" O sistema aplica imediatamente as informações em toda a interface.
  5. Confira se os dados aparecem corretamente Verifique na barra lateral, no dashboard e na tela de login.
Dica: Use o botão "Pré-visualizar" para ver como ficará antes de salvar. Isso evita erros de digitação que apareceriam para todos os usuários.
Seção 7

Cadastrar o Primeiro Usuário do Cliente

Agora você vai criar o usuário definitivo do cliente — a pessoa que vai administrar o sistema no dia a dia.

  1. No menu lateral, clique em "Usuários & Acessos"
  2. Clique em "Novo Usuário"
  3. Preencha os dados do usuário Nome, login (e-mail ou usuário) e senha. Defina a role como master para que ele tenha acesso completo ao sistema.
  4. Salve o usuário
  5. Informe as credenciais ao responsável do cliente Anote em papel e entregue pessoalmente. Oriente-o a trocar a senha no primeiro acesso.
Após criar o usuário definitivo do cliente, exclua ou desative o usuário admin padrão para garantir a segurança do sistema.
Seção 8

Testar o Sistema

Antes de encerrar a visita, execute o checklist abaixo para garantir que tudo está funcionando corretamente.

  • O sistema abre sem erros no navegador ao clicar em ABRIR.bat
  • O login funciona com o usuário criado para o cliente
  • O nome e o logo da empresa aparecem corretamente na interface
  • É possível cadastrar um colaborador de teste sem erros
  • A Folha de Ponto carrega e calcula corretamente para o colaborador de teste
  • A impressão da folha de ponto exibe os dados corretos da empresa (nome, CNPJ, ramo de atividade)
Se todos os itens estão marcados, a implantação foi concluída com sucesso! Lembre-se de excluir o colaborador de teste antes de entregar o sistema ao cliente.
Seção 9

Criar Atalho na Área de Trabalho (opcional)

Para facilitar o acesso diário do cliente ao sistema, crie um atalho do ABRIR.bat na área de trabalho.

  1. Abra a pasta onde os arquivos foram instalados Por exemplo: C:\DRG-Kronos\
  2. Clique com o botão direito no arquivo ABRIR.bat
  3. Selecione "Enviar para" → "Área de Trabalho (criar atalho)" Um atalho aparece na área de trabalho.
  4. Renomeie o atalho para "DRG-Kronos 3.0" Clique com o botão direito no atalho → "Renomear". Digite o nome e pressione Enter.
A partir de agora, o cliente pode abrir o sistema com um duplo clique no atalho da área de trabalho — sem precisar navegar pelas pastas.
Seção 10

Limitações da Versão Local (Pen Drive / Arquivo)

A versão instalada localmente a partir do pen drive funciona muito bem para o uso diário de gestão. Porém, algumas funcionalidades dependem do acesso via internet.

Requer conexão com a internet

O sistema precisa estar conectado à internet para acessar o Firebase Firestore. Sem internet, nenhum dado é carregado ou salvo. Isso é normal e esperado — o banco de dados fica na nuvem.

FuncionalidadeVersão LocalSolução
Gestão de colaboradores, folha de ponto, relatórios Funciona normalmente —
Leitura de PDF por IA (Gemini) Não disponível Funciona apenas na versão hospedada na web (GitHub Pages)
App de Ponto (ponto.html) pelo celular Não funciona via file:// no celular O arquivo precisa ser acessado via URL na internet. Hospedar no GitHub Pages resolve (gratuito)
Solução para funcionalidade completa: Hospedar o sistema no GitHub Pages (gratuito) disponibiliza todas as funcionalidades, incluindo a leitura de PDF por IA e o App de Ponto pelo celular. Para configurar a hospedagem, consulte o suporte técnico da D.R. Global.
